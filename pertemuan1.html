🍼 Level: Bener-bener dari NOL

Belajar SQL, sepelan ngajarin bayi ngomong kata pertama.

SQL itu bukan bahasa buat bikin tampilan, tapi bahasa buat ngobrol sama database — nanya, nyuruh nyimpen, nyuruh ubah, nyuruh hapus data. Kalau kamu sudah ikutin seri belajar PHP sebelumnya, beberapa kata di sini bakal terasa familiar. Kita bahas satu-satu, pelan-pelan, sampai kamu bisa "ngobrol" lancar sama database sendiri. 🐢

Bab 01

Apa itu SQL? 🤔

Sebelum ngobrol sama database, kenalan dulu sama bahasanya.

🗣️ Bayangin gini
Database itu kayak petugas gudang arsip raksasa yang nyimpen jutaan data rapi-rapi. Masalahnya, petugas ini cuma ngerti satu bahasa khusus. Nah, SQL (Structured Query Language) adalah bahasa itu — cara kamu "ngomong" ke petugas gudang: "tolong ambilkan data ini", "tolong simpan data baru ini", "tolong buang data yang itu".

SQL dipakai untuk empat hal utama, yang nanti bakal jadi sahabat karibmu:

PerintahArtinya
SELECT"Tolong ambilkan / tampilkan data ini"
INSERT"Tolong simpan data baru ini"
UPDATE"Tolong ubah data yang ini"
DELETE"Tolong buang data yang ini"
💡 Kata Kak SQL
Kalau kamu sudah pernah bikin tabel siswa di seri belajar PHP sebelumnya lewat phpMyAdmin, kamu sebenarnya sudah pernah pakai SQL — cuma waktu itu belum dijelasin detail. Sekarang kita bongkar pelan-pelan.

Kamu bisa coba semua kode di halaman ini lewat phpMyAdmin (buka http://localhost/phpmyadmin, pilih database, klik tab SQL, tempel kodenya, klik "Go").

Bab 02

Tabel & kolom — kayak Excel, tapi lebih rapi 📊

Sebelum ngobrol, pahami dulu bentuk "kamarnya".

📊 Bayangin gini
Database itu lemari besar. Di dalamnya ada beberapa laci (disebut tabel). Tiap laci punya kolom-kolom tetap (nama, umur, kelas), dan tiap kali kamu masukin satu data baru, itu jadi satu baris baru di laci itu — persis kayak spreadsheet Excel.

Contoh tabel bernama siswa:

idnamakelasnilai
1BudiXII RPL88.5
2SintaXI TKJ92.0
  • Tabel: siswa — laci penyimpanan data siswa.
  • Kolom: id, nama, kelas, nilai — jenis informasi yang disimpan tiap siswa.
  • Baris: tiap 1 baris = data 1 siswa.
💡 Kata Kak SQL
Satu database bisa punya banyak tabel sekaligus — misalnya database toko bisa punya tabel produk, pelanggan, dan transaksi secara bersamaan.

Bab 03

Bikin tabel pertamamu (CREATE TABLE) 🧱

Nyusun laci kosong sebelum diisi barang.

bikin-tabel.sql
CREATE TABLE siswa (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nama VARCHAR(100),
  kelas VARCHAR(50),
  nilai FLOAT
);
🔧 Pelan-pelan kita bedah
  • CREATE TABLE siswa artinya "buatkan laci baru bernama siswa".
  • Di dalam kurung, tiap baris adalah satu kolom: nama kolom, lalu jenis datanya.
  • id INT AUTO_INCREMENT PRIMARY KEY: kolom id berisi angka, otomatis nambah sendiri, dan jadi "nomor identitas unik" tiap baris — nggak akan ada dua siswa dengan id yang sama.
  • VARCHAR(100): kolom tulisan, maksimal 100 huruf.
  • FLOAT: kolom angka desimal.
Tipe dataBuat nyimpen apa
INTAngka bulat (1, 2, 100, dst)
FLOAT / DECIMALAngka desimal (88.5, 12.75)
VARCHAR(n)Tulisan pendek, maksimal n huruf
TEXTTulisan panjang, tanpa batas ketat
DATETanggal (contoh: 2026-07-13)
BOOLEANBenar / salah (1 atau 0)
⚠️ Jangan lupa titik koma
Hampir setiap perintah SQL diakhiri tanda titik koma ; — kayak titik di akhir kalimat. Kadang boleh kelupaan (SQL cukup toleran), tapi biasakan nulis biar rapi dan nggak bingung sendiri nanti.

Bab 04

Masukin data baru (INSERT) 📥

Sekarang laci kosong tadi mau kita isi barang.

insert.sql
INSERT INTO siswa (nama, kelas, nilai)
VALUES ('Budi Santoso', 'XII RPL', 88.5);
📦 Pelan-pelan kita bedah
  • INSERT INTO siswa artinya "masukkan ke laci siswa".
  • (nama, kelas, nilai): kolom mana saja yang mau diisi (kolom id nggak perlu ditulis, karena otomatis terisi sendiri).
  • VALUES (...): nilai yang mau dimasukkan, urutannya harus sama persis dengan urutan kolom di atas.
  • Tulisan diapit tanda kutip satu ' ', angka tidak perlu tanda kutip.

Mau masukin beberapa data sekaligus? Bisa banget:

insert-banyak.sql
INSERT INTO siswa (nama, kelas, nilai) VALUES
('Sinta Dewi', 'XI TKJ', 92.0),
('Doni Pratama', 'X MM', 79.0);

Bab 05

Lihat data (SELECT) 👀

Perintah yang paling sering kamu pakai, sesering kamu bernapas.

select.sql
SELECT * FROM siswa;
👀 Pelan-pelan kita bedah
SELECT * FROM siswa artinya "ambil semua kolom (tanda bintang * artinya "semua"), dari laci siswa". Hasilnya adalah semua baris data siswa, dengan semua kolomnya.

Kalau cuma butuh kolom tertentu saja, sebutkan namanya:

select-kolom.sql
SELECT nama, nilai FROM siswa;
✅ Hasilnya
namanilai
Budi Santoso88.5
Sinta Dewi92.0
Doni Pratama79.0
💡 Kata Kak SQL
Kebiasaan yang baik: jangan selalu pakai * kalau nggak semua kolom dibutuhkan. Selain hasilnya lebih rapi, ini juga bikin database kerja lebih ringan karena nggak perlu ambil data yang nggak dipakai.

Bab 06

Filter data pakai WHERE 🔍

Biar nggak semua data ikut kebawa, cuma yang kamu mau aja.

🔍 Bayangin gini
Kalau SELECT tanpa WHERE itu kayak nyuruh petugas gudang "ambil semua barang di laci ini", WHERE itu kayak nambahin syarat: "ambil yang ini aja, yang cocok sama syarat tertentu".
where.sql
SELECT * FROM siswa WHERE kelas = 'XII RPL';

Bisa juga pakai operator perbandingan buat angka:

where-angka.sql
SELECT * FROM siswa WHERE nilai >= 85;
OperatorArtinya
=Sama dengan
!= atau <>Tidak sama dengan
> / <Lebih besar / lebih kecil
>= / <=Lebih besar sama dengan / lebih kecil sama dengan
💡 Kata Kak SQL
WHERE juga dipakai di UPDATE dan DELETE nanti — dan di situ, WHERE jadi sangat penting. Lupa nulis WHERE saat hapus data bisa berakibat semua data terhapus. Nanti kita bahas lebih detail.

Bab 07

Urutkan (ORDER BY) & batasi (LIMIT) 📐

Biar data ditampilkan rapi dan nggak kepanjangan.

order-limit.sql
SELECT * FROM siswa
ORDER BY nilai DESC
LIMIT 2;
📐 Pelan-pelan kita bedah
  • ORDER BY nilai: urutkan hasilnya berdasarkan kolom nilai.
  • DESC (descending) artinya dari besar ke kecil. Kalau mau dari kecil ke besar, pakai ASC (ascending) — atau tidak usah ditulis sama sekali, karena ASC itu default-nya.
  • LIMIT 2: cuma tampilkan 2 baris teratas saja dari hasilnya. Cocok banget buat, misalnya, "tampilkan 2 siswa dengan nilai tertinggi".
💡 Kata Kak SQL
Urutan penulisan klausa SQL itu penting: SELECT ... FROM ... WHERE ... ORDER BY ... LIMIT .... Kalau dibolak-balik urutannya, SQL akan error.

Bab 08

Ubah data (UPDATE) ✏️

Data lama boleh berubah, nggak selamanya harus dihapus terus dibikin ulang.

update.sql
UPDATE siswa
SET nilai = 95.0
WHERE nama = 'Budi Santoso';
✏️ Pelan-pelan kita bedah
  • UPDATE siswa: "yang mau diubah ada di laci siswa".
  • SET nilai = 95.0: "ubah kolom nilai jadi 95.0".
  • WHERE nama = 'Budi Santoso': "tapi cuma buat baris yang namanya Budi Santoso aja".
🚨 INI PALING PENTING DI SELURUH SQL
Kalau kamu tulis UPDATE siswa SET nilai = 95.0; tanpa WHERE, maka SEMUA baris di tabel siswa nilainya berubah jadi 95.0 — bukan cuma satu siswa. Selalu double-check bagian WHERE sebelum menjalankan perintah UPDATE.
💡 Kata Kak SQL
Kebiasaan aman: coba dulu pakai SELECT * FROM siswa WHERE nama = 'Budi Santoso' untuk lihat data mana yang bakal kena — kalau hasilnya sudah benar (cuma 1 baris yang dimaksud), baru ganti SELECT * jadi UPDATE ... SET ....

Bab 09

Hapus data (DELETE) 🗑️

Perintah paling pendek, tapi paling perlu hati-hati sedunia.

delete.sql
DELETE FROM siswa
WHERE nama = 'Doni Pratama';
🚨 Sekali lagi, jangan lupa WHERE
DELETE FROM siswa; tanpa WHERE akan menghapus SELURUH data di tabel siswa, dan itu tidak bisa dibatalkan kecuali kamu punya backup. Ini kesalahan klasik yang bikin banyak pemula (bahkan yang sudah kerja beberapa tahun!) panik mendadak. Selalu cek WHERE-nya dua kali sebelum tekan enter.
💡 Kata Kak SQL
Sama seperti UPDATE, biasakan cek dulu pakai SELECT dengan WHERE yang sama, pastikan yang muncul memang cuma data yang mau dihapus, baru ganti jadi DELETE FROM.

Bab 10

Filter yang lebih pintar (AND, OR, LIKE, IN, BETWEEN) 🧠

WHERE dasar sudah oke, sekarang bikin dia lebih jago.

Kata kunciGunanyaContoh
ANDSemua syarat harus terpenuhiWHERE kelas='XII RPL' AND nilai>85
ORSalah satu syarat cukupWHERE kelas='X MM' OR kelas='XI TKJ'
LIKECari yang mirip / mengandung teksWHERE nama LIKE '%budi%'
INCocok dengan salah satu dari daftarWHERE kelas IN ('X MM','XI TKJ')
BETWEENAda di antara dua nilaiWHERE nilai BETWEEN 80 AND 90
filter-lanjut.sql
-- cari siswa yang namanya mengandung kata "budi"
SELECT * FROM siswa WHERE nama LIKE '%budi%';
🔎 Soal si tanda persen %
Di LIKE, tanda % artinya "apapun boleh ada di sini". Jadi '%budi%' artinya "ada tulisan 'budi' di mana pun posisinya", 'budi%' artinya "harus diawali 'budi'", dan '%budi' artinya "harus diakhiri 'budi'".

Bab 11

Hitung-hitungan otomatis (fungsi agregat) 🧮

Biar nggak perlu hitung manual satu-satu.

FungsiGunanya
COUNT()Menghitung jumlah baris
SUM()Menjumlahkan semua nilai di satu kolom
AVG()Menghitung rata-rata
MAX()Mencari nilai tertinggi
MIN()Mencari nilai terendah
agregat.sql
SELECT COUNT(*) AS jumlah_siswa, AVG(nilai) AS rata_rata
FROM siswa;
💡 Kata Kak SQL
AS dipakai buat kasih "nama panggilan" pada hasil, biar lebih gampang dibaca — COUNT(*) AS jumlah_siswa artinya "hasil hitungan ini, tampilkan dengan nama kolom jumlah_siswa".

Kalau mau menghitung per kelompok (misalnya rata-rata nilai per kelas), pakai GROUP BY:

group-by.sql
SELECT kelas, AVG(nilai) AS rata_rata
FROM siswa
GROUP BY kelas;
🗂️ Bayangin gini
GROUP BY kelas itu kayak menyortir siswa ke tumpukan-tumpukan terpisah berdasarkan kelasnya dulu, baru dihitung rata-rata di masing-masing tumpukan secara terpisah — bukan dicampur jadi satu rata-rata besar.

Bab 12

Primary Key & Foreign Key — cara tabel saling kenal 🔑

Konsep penting sebelum belajar menggabungkan tabel.

🪪 Bayangin gini
Primary Key itu kayak Nomor Induk Siswa (NIS) — nomor unik yang nggak mungkin sama antara satu siswa dengan siswa lain, dan jadi "identitas resmi" tiap baris data.
🔗 Bayangin gini (lanjutan)
Foreign Key itu kayak nomor NIS yang dicatat ulang di buku perpustakaan saat siswa pinjam buku. Nomor itu "menunjuk balik" ke data siswa yang lengkap di tabel siswa, tanpa perlu menulis ulang nama, kelas, dsb di buku peminjaman. Ini yang bikin dua tabel bisa "saling kenal".

Contoh: dua tabel yang berhubungan.

relasi.sql
CREATE TABLE siswa (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nama VARCHAR(100)
);

CREATE TABLE peminjaman (
  id INT AUTO_INCREMENT PRIMARY KEY,
  siswa_id INT,           -- ini foreign key
  judul_buku VARCHAR(150),
  FOREIGN KEY (siswa_id) REFERENCES siswa(id)
);
💡 Kata Kak SQL
Kolom siswa_id di tabel peminjaman cuma nyimpen angka (misalnya 1), yang menunjuk ke siswa dengan id = 1 di tabel siswa. Ini jauh lebih hemat dan rapi dibanding menulis ulang nama & kelas siswa di setiap baris peminjaman.

Bab 13

Menggabungkan tabel (JOIN) 🧩

Setelah dua laci "saling kenal", sekarang kita lihat isinya bareng-bareng.

🧩 Bayangin gini
Kalau tabel peminjaman cuma nyimpen siswa_id (angka doang), itu nggak enak dibaca manusia. JOIN itu kayak menyatukan dua lembar kertas terpisah jadi satu tampilan: data peminjaman digabung dengan data siswa aslinya, biar yang muncul nama siswa, bukan cuma angka id.
join.sql
SELECT siswa.nama, peminjaman.judul_buku
FROM peminjaman
JOIN siswa ON peminjaman.siswa_id = siswa.id;
🔍 Pelan-pelan kita bedah
  • FROM peminjaman: mulai dari tabel peminjaman.
  • JOIN siswa: gabungkan dengan tabel siswa.
  • ON peminjaman.siswa_id = siswa.id: "caranya gabungin, cocokkan kolom siswa_id di peminjaman dengan kolom id di siswa" — ini aturan yang bilang baris mana harus disatukan dengan baris mana.
  • Hasilnya: satu tabel gabungan yang langsung nampilin nama siswa dan judul buku yang dipinjam dalam satu baris, tanpa perlu lihat angka id sama sekali.
💡 Kata Kak SQL
JOIN ini salah satu skill SQL paling powerful. Hampir semua sistem informasi beneran (seperti project Sistem Informasi Data Siswa yang pernah kamu bikin) akan butuh JOIN begitu datanya sudah dipecah jadi beberapa tabel berhubungan.

Bab 14

Studi kasus: Database Toko Kecil 🏪

Gabungin semua yang sudah dipelajari jadi satu database mini yang nyata.

Kita bikin database toko kecil dengan dua tabel: produk dan penjualan.

toko.sql
CREATE TABLE produk (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nama_produk VARCHAR(100),
  harga INT
);

CREATE TABLE penjualan (
  id INT AUTO_INCREMENT PRIMARY KEY,
  produk_id INT,
  jumlah INT,
  FOREIGN KEY (produk_id) REFERENCES produk(id)
);

INSERT INTO produk (nama_produk, harga) VALUES
('Roti Coklat', 8000),
('Kopi Susu', 12000);

INSERT INTO penjualan (produk_id, jumlah) VALUES
(1, 3),
(2, 5);

Sekarang, coba cari tahu: total pendapatan dari tiap produk.

laporan.sql
SELECT produk.nama_produk,
       penjualan.jumlah,
       (produk.harga * penjualan.jumlah) AS total
FROM penjualan
JOIN produk ON penjualan.produk_id = produk.id;
✅ Hasilnya
nama_produkjumlahtotal
Roti Coklat324000
Kopi Susu560000
🎉 Selamat!
Query di atas menggabungkan hampir semua yang sudah kamu pelajari: dua tabel yang berelasi, JOIN, perhitungan manual di dalam SELECT, dan pemberian nama kolom pakai AS. Ini persis pola yang dipakai laporan penjualan di toko-toko online sungguhan.

Bab 15

Sudah sampai sini, lanjut ke mana? 🌱

Kamu sekarang sudah bisa "ngobrol" beneran sama database.

🏁 Yang sudah kamu kuasai
Membuat tabel, memasukkan data, menampilkan & menyaring data, mengurutkan & membatasi hasil, mengubah & menghapus data dengan aman, filter lanjutan (AND/OR/LIKE/IN/BETWEEN), fungsi hitung otomatis (COUNT/SUM/AVG), sampai menggabungkan tabel dengan JOIN. Ini adalah fondasi SQL yang lengkap untuk kebutuhan sehari-hari.

Langkah selanjutnya yang disarankan:

  1. Balik lagi ke project Sistem Informasi Data Siswa dari seri PHP, coba modifikasi query-nya pakai teknik yang baru kamu pelajari di sini (misalnya tambahkan pencarian pakai LIKE).
  2. Coba bikin sendiri database dengan topik lain, misalnya database perpustakaan (buku, anggota, peminjaman) lengkap dengan relasi antar tabelnya.
  3. Pelajari jenis-jenis JOIN lain (LEFT JOIN, RIGHT JOIN) setelah nyaman dengan JOIN dasar.
  4. Pelajari INDEX untuk mempercepat pencarian data ketika tabelnya sudah berisi ribuan baris.
💬 Pesan terakhir dari Kak SQL
SQL itu salah satu skill yang paling "abadi" di dunia teknologi — dipakai di hampir semua aplikasi, dari yang kecil sampai yang raksasa. Begitu kamu paham SELECT, WHERE, dan JOIN dengan baik, kamu sudah punya modal yang dipakai jutaan developer di seluruh dunia setiap hari. Pelan-pelan aja, kamu sudah jalan jauh. 🌟
Belajar SQL dari Nol — pelengkap seri "Belajar PHP dari Nol", pelan-pelan aja, nggak perlu buru-buru. 🐣
